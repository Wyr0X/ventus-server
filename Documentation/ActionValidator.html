Resumen del Sistema de Acciones

Descripción general del sistema de acciones, helpers y validadores

Introducción

El sistema de acciones está diseñado para manejar las interacciones del jugador con el mundo del juego, como lanzar hechizos, atacar, usar ítems o equipar objetos. Utiliza validadores, helpers y objetos específicos para garantizar que las acciones sean válidas antes de ejecutarse.

1. Componentes Principales

1.1. Validadores de Acciones (IActionValidator)

Los validadores son responsables de verificar si una acción puede ser intentada o ejecutada. Cada tipo de acción tiene su propio validador que implementa la interfaz IActionValidator.

public interface IActionValidator
{
    ValidationResult CanAttemptAction(PlayerObject player, object target = null, SpellObject spell = null);
    ValidationResult CanExecuteAction(PlayerObject player, object target = null, SpellObject spell = null);
}

Ejemplo: SpellActionValidator

public class SpellActionValidator : IActionValidator
{
    private readonly SpellObject _spell;

    public SpellActionValidator(SpellObject spell)
    {
        _spell = spell;
    }

    public ValidationResult CanAttemptAction(PlayerObject player, object target = null, SpellObject spell = null)
    {
        if (_spell == null)
            return ValidationResult.Fail("Hechizo no válido.");

        if (!ActionValidationHelper.CanCastSpell(player, _spell))
            return ValidationResult.Fail("No puedes lanzar este hechizo.");

        return ValidationResult.Success();
    }

    public ValidationResult CanExecuteAction(PlayerObject player, object target = null, SpellObject spell = null)
    {
        if (_spell == null)
            return ValidationResult.Fail("Hechizo no válido.");

        return ValidationResult.Success();
    }
}

1.2. Helpers de Validación

Los helpers son funciones auxiliares que encapsulan lógica común para validaciones. Se dividen en:

  • Helpers Comunes: Validaciones generales aplicables a todas las acciones.
  • Helpers para Intentar Acciones: Validaciones previas al intento de ejecutar una acción.
  • Helpers para Ejecutar Acciones: Validaciones específicas para la ejecución de una acción.

Helpers Comunes

public static class ActionValidationHelper
{
    public static bool CanPerformActions(PlayerObject player)
    {
        return !player.Stats.IsDead
            && !player.Stats.IsStunned
            && !player.Stats.IsRooted
            && !player.Stats.IsSilenced
            && !player.Stats.IsChanneling;
    }

    public static bool IsTargetValid(object target)
    {
        if (target is PlayerObject targetPlayer)
        {
            return !targetPlayer.Stats.IsDead && targetPlayer.IsOnline;
        }
        return false;
    }

    public static bool HasSufficientResources(PlayerObject player, int manaCost, int energyCost)
    {
        return player.Stats.Mana >= manaCost && player.Stats.Energy >= energyCost;
    }
}

Helpers para Intentar Acciones

public static class AttemptActionHelper
{
    public static ValidationResult CanAttemptCastSpell(PlayerObject player, SpellObject spell)
    {
        if (!ActionValidationHelper.CanPerformActions(player))
            return ValidationResult.Fail("No puedes realizar acciones en este momento.");

        if (!ActionValidationHelper.HasSufficientResources(player, spell.Model.ManaCost, 0))
            return ValidationResult.Fail("No tienes suficiente maná para lanzar este hechizo.");

        return ValidationResult.Success();
    }
}

Helpers para Ejecutar Acciones

public static class ExecuteActionHelper
{
    public static ValidationResult CanExecuteCastSpell(PlayerObject player, SpellObject spell, object target)
    {
        if (!ActionValidationHelper.IsTargetValid(target))
            return ValidationResult.Fail("El objetivo no es válido.");

        return ValidationResult.Success();
    }
}

2. Objetos Validables (IValidatableObject)

Objetos como hechizos, ítems y equipos implementan la interfaz IValidatableObject para asociar un validador específico.

Definición de la Interfaz

public interface IValidatableObject
{
    IActionValidator GetValidator();
}

Ejemplo: SpellObject

public class SpellObject : GameObject, IValidatableObject
{
    public SpellModel Model { get; }
    public float CurrentCooldown { get; private set; } = 0f;

    public SpellObject(string id, Vec2 position, SpellModel model)
        : base(id, position)
    {
        Model = model ?? throw new ArgumentNullException(nameof(model));
    }

    public IActionValidator GetValidator()
    {
        return new SpellActionValidator(this);
    }
}

3. Flujo General del Sistema

  1. Recepción del Paquete: Validar el paquete y determinar el tipo de acción (hechizo, golpe, ítem, equipar).
  2. Intentar la Acción: Usar el validador asociado al objeto para verificar si la acción puede ser intentada.
  3. Ejecutar la Acción: Usar el validador para verificar si la acción puede ser ejecutada y aplicar sus efectos.
  4. Logs y Notificaciones: Generar logs de combate y enviar un paquete CombatInfo a los jugadores involucrados.
